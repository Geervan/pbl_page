Prioritising of GUI Regression Testcases using Agentic AI

"Hypothesis: Multi-Agent systems can leverage semantic action-metadata and distilled history to overcome the limitations of classic rule-based TCP, achieving higher fault detection rates in GUI environments."

Problem Statement

Traditional Test Case Prioritisation (TCP) relies on static heuristics like code coverage or history. However, in Modern GUI-heavy applications, these methods fail to understand the semantic intent of UI changes. This project introduces an Agentic AI approach to intelligently rank tests based on visual risk and historical failure signals.

The Planner

Analyzes the change summary to formulate a testing strategy. Decides if a "Smoke Test" or "Deep Regression" is required based on the blast radius of the commit.

The Critic

Challenges the initial ranking. It looks for "Logical Blindspots"—ensuring that secondary dependencies (like a shared footer change) aren't ignored by the primary ranker.

The Explainer

Generates human-readable rationales for why a specific test was ranked #1. This builds trust with developers by explaining why the AI considers a test high-risk.

Results & Analysis

Typical APFD Score 0.7833

The Agentic AI model consistently outperforms random baselines. In "Cold Start" scenarios (brand new code changes), it achieves an APFD of ~0.78, proving semantic reasoning. After "Distilled Memory" is engaged, accuracy trends upwards as the system identifies historical failure patterns.
